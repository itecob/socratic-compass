## Process note — Cowork-side file write reliability

During Phase 2 polish, three Edit-tool calls to existing files in `.architecture/` silently failed to persist (bootstrap script truncated mid-line; ADR 0009 and 0011 addenda not applied; manifest/invariants/debt-log appends not applied). All were re-applied via bash heredoc, which persisted reliably. The Edit tool's failure mode on the mounted Windows folder appears to be: Edit reports success, in-memory Read shows the new content, but disk write doesn't flush. Workaround: use bash heredocs for modifications to files in the mounted folder going forward. Logging as a process note here rather than DEBT because it's a tooling issue, not a project debt.
