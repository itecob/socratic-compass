# Premise-Check — 2026-06-26 19:00

**Source:** `.architecture/interviews/2026-06-26-1830.md`
**Skill:** `compass:premise-check`

## Output 1 — Problem framing

`compass:using-compass`, when invoked in a project without `.architecture/`, currently prints a *recommendation* to run `bootstrap-architecture.sh` and then proceeds passively. This trains users to ignore the recommendation. INV-002 verification on the second computer caught the failure mode. The fix: an assertive consent-framed wizard that owns the onboarding sequence end-to-end. The wizard's frame is "informed-consent permission-unlock," not "feature-discovery." Click-through is acceptable as a deliberate release of control. Compass owns the wizard, not the platform's per-write consent prompts.

## Output 2 — Load-bearing assumptions

1. **Compass owns the onboarding** (not deferred to Cowork/Claude Code per-write prompts).
2. **Frame is informed-consent**, not feature discovery. The wizard's purpose is permission-unlock.
3. **Click-through is acceptable** when it's a deliberate release of control. "Just build it" is a valid path; default-to-agent-takes-action is not.
4. **`.architecture/` existence is the consent record.** No separate state file. Wizard runs only when `.architecture/` is missing; once it exists, wizard never re-runs.
5. **Single host:** `compass:using-compass` is the only skill that runs the wizard. Other Compass skills detect missing `.architecture/` and error-and-redirect to using-compass.
6. **Post-wizard auto-pivot via active-context review.** The agent re-reads recent conversation turns (heuristic: last 10) and resumes the user's original intent. If ambiguous, asks "you wanted X, right?" before pivoting.
