### Concerns raised

1. **`set -eu` near-miss with INV-003.** All hooks use `set -eu`. The `date -r` arithmetic on `session-end.sh:22` could abort with non-zero exit on BSD/macOS or if file disappears mid-run. INV-003 passes today, but the failure mode is one missing guard away.
2. **`pre-tool-use-write.sh` header comment** still said "for Edit" (file is a copy of edit per Task 34).
3. **Broken `session_start_sha` heuristic.** `git log --reverse --format=%H -n 1` returns the OLDEST commit, not the session start. The "no commits this session" check almost always fires on young repos, training users to ignore the hook.
4. **`session-end.sh` phase-validation reminder** (lines 19–28) invents discipline at hook layer that no skill enforces. Violates ADR 0017's "reinforcement, not source" boundary.
5. **SessionStart "recent activity" listing** is scope creep over Task 32 but reasonable (single line per dir, only when something exists).
6. **Debt-log grep is unanchored** — matches debt entries when `$TARGET` appears anywhere in the `Files:` line. Could produce false positives on short or common substrings.
7. **Cowork-side parity gap:** the SessionStart "latest premise-check/validation listing" has no skill-body counterpart. Cowork users miss that reinforcement.
